Problem Analysis

This problem asks me to solve Pell's equation, x² - Dy² = 1, for every non-square integer D up to 1000, and to report the D whose minimal (fundamental) solution has the largest x. The phrase "minimal solution in x" means the smallest positive-integer pair (x, y) satisfying the equation; every other solution is generated from it. The naive idea - iterate over candidate values of x or y and test - is hopeless, because the fundamental solution can be astronomically large. For D = 661 the value of x exceeds 10^17, so any search bounded by the magnitude of the answer would never terminate. The problem only becomes tractable through a classical theorem of number theory: the fundamental solution of Pell's equation always appears among the continued fraction convergents of sqrt(D).

A continued fraction expresses a number as a0 + 1/(a1 + 1/(a2 + ...)), and a convergent is the rational p/q obtained by truncating that expansion at a chosen depth. For a quadratic irrational such as sqrt(D), the expansion is eventually periodic - the integer terms repeat in a fixed-length cycle - which is exactly what lets me compute the whole thing in finite time. The crucial refinement is that one specific convergent yields the fundamental solution, and which one depends on the parity of the period length L. If L is odd, the solution sits at convergent index L - 1; if L is even, at index 2L - 1. The period length is O(sqrt(D)), and the numerators of the convergents grow exponentially with the index, so the digits of x grow as O(sqrt(D)) too - which forces me to use arbitrary-precision arithmetic. The overall cost is O(D * sqrt(D)) big-integer operations, fast enough to finish in milliseconds for D up to 1000.

Solution Approach

Continued-Fraction Convergents for Pell's Equation (Python and C)

Both files implement the same algorithm; Python defers big-integer arithmetic to its runtime, while C carries its own big-integer type. I chose this method because it is correct by theorem and runs in time proportional to the period length rather than the magnitude of the answer.

Step 1 - Generate the continued fraction. Starting from a0 = floor(sqrt(D)), I produce each subsequent term with the standard recurrence over three state variables m, n, a: m = n*a - m, n = (D - m*m)/n, a = floor((sqrt(D) + m)/n). These three integers fully encode the "reduced" form of the quadratic surd at each step. I stop when a term equals 2*a0, which is the classical signal that one full period of the continued fraction has been emitted. In C this lives in compute_cf; in Python it is the while loop inside find_fundamental_solution_to_pell_equation.

Step 2 - Pick the right convergent index. With the stored array length including a0, the period length is cf_len - 1. If that length is even I evaluate the convergent at index 2*cf_len - 3; if odd, at index cf_len - 2. This parity rule comes directly from Pell theory and means I never have to generate or test more than the one convergent that matters.

Step 3 - Evaluate the convergent backward. I evaluate the n-th convergent from the innermost term outward, repeatedly applying term + 1/running_value. To reach indices beyond a single period I wrap the term index with (i - 1) % period_length + 1, holding a0 fixed at position 0 and cycling through the periodic part. The backward evaluation keeps only two big integers (numerator and denominator) alive at any moment, which matters because the convergent index can be large.

Language-specific differences. In Python I use fractions.Fraction, which carries an exact arbitrary-precision numerator and denominator and reduces to lowest terms automatically. In C, where there is no native big integer, I implement a BigInt struct holding up to 512 limbs in base 10^9, little-endian. The only operations I need are addition (bi_add) and multiplication by a small scalar (bi_mul_small); both are simple schoolbook loops. I chose base 10^9 because a limb fits comfortably in a 64-bit word and the product of two limbs fits in 128 bits, which I obtain through GCC's unsigned __int128 extension.
