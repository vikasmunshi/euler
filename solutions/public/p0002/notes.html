Problem Analysis

The Fibonacci sequence is defined by the recurrence relation F(n) = F(n-1) + F(n-2). Here the sequence is seeded with 1 and 2, and the task is to sum every even-valued term that does not exceed four million. The naive approach generates every Fibonacci number and tests each for evenness, but a structural observation about parity removes the test entirely. Writing out the parity of the terms - 1 (odd), 2 (even), 3 (odd), 5 (odd), 8 (even), 13 (odd), 21 (odd), 34 (even) - shows that every third term is even. This follows directly from the addition rule on parity: odd+even=odd, even+odd=odd, odd+odd=even, so the pattern repeats with period three forever.

Knowing the even terms recur every three steps, I can derive a recurrence that steps directly from one even Fibonacci number to the next, skipping the two odd ones between. If E(k) is the k-th even Fibonacci number, then E(k+1) = 4*E(k) + E(k-1), which is verifiable by expanding the standard recurrence three times. The payoff is that I need only two integer state variables and a single multiply-add per step, with no divisibility test. Because Fibonacci numbers grow exponentially - each term is roughly 1.618 times the previous, the golden ratio - the count of terms below any limit L is O(log L). The algorithm therefore runs in logarithmic time and O(1) space, finishing in a handful of iterations even for a limit of four million.

Solution Approach

Direct Even-Fibonacci Recurrence (Python and C)

Both files implement the same algorithm. I keep two consecutive even Fibonacci numbers, even_fib_a and even_fib_b, seeded with 2 and 8 - the first two even Fibonacci terms. Each step accumulates even_fib_a into the running total, then advances the pair with next = 4 * even_fib_b + even_fib_a, shifting even_fib_a = even_fib_b and even_fib_b = next. The loop exits as soon as even_fib_a reaches or exceeds the limit, so the limit is never included - the "does not exceed" bound is applied strictly before accumulation.

I chose the seed (2, 8) rather than deriving it from first principles because both values are easy to confirm by inspection against the listed sequence, and a recurrence is only as correct as its initial conditions. A wrong seed such as (0, 2) would silently corrupt every later term, since 0 is not part of the stated sequence.

The Python version wraps the iteration in a private generator _even_fibonacci_numbers and feeds it to sum(), cleanly separating sequence generation from aggregation. The C version is a plain while loop accumulating into a long long to guard against overflow. Both delegate argument parsing to the runner framework's parse_int helper, so the remaining difference is purely idiomatic: a lazy generator feeding sum() in Python versus an explicit accumulation loop in C. Both produce 4613732 for the four-million case; the recorded averages are about 0.42 microseconds in C and 1.6 microseconds in Python.

Programming Techniques

Generator Functions for Lazy Sequences

A Python generator uses yield to produce values one at a time without materialising a list. Here _even_fibonacci_numbers emits each even term as it is computed and stops when the term reaches the limit, holding only its two state variables between yields. Pairing it with sum() keeps generation and aggregation decoupled and readable. Reach for a generator whenever a sequence is conceptually open-ended or its length is unknown in advance.

Tuple (Parallel Assignment) for State Advancement

The line even_fib_a, even_fib_b = (even_fib_b, 4 * even_fib_b + even_fib_a) uses parallel assignment: the entire right-hand side is evaluated before either name is bound, so the old even_fib_a is still available when computing the new even_fib_b. This avoids a temporary variable. The C code achieves the same effect with an explicit next local, which maps directly onto how a register machine executes the update. Use parallel assignment when you must update interdependent variables simultaneously without introducing ordering bugs.

Two-Variable Rolling State for Linear Recurrences

Any depth-2 linear recurrence X(n) = a*X(n-1) + b*X(n-2) can be computed with exactly two rolling state variables, giving O(1) space. This sliding-window pattern recurs constantly: Fibonacci variants, Lucas numbers, tribonacci sequences, and many dynamic programming transitions. Recognising it lets you avoid storing the full history.

Exploiting Mathematical Structure to Eliminate Filtering

Instead of generating all Fibonacci numbers and testing each with n % 2 == 0, I derived a recurrence that produces only the even terms. This replaces a branch-laden loop with a branch-free one that runs about three times shorter. When a problem asks for a regularly patterned subsequence, deriving a direct recurrence on that subsequence is almost always cleaner and faster than filtering.

Key Takeaways

The unlocking insight is that the even Fibonacci numbers form their own Fibonacci-like sequence governed by E(k+1) = 4*E(k) + E(k-1). This collapses the problem to a two-variable loop with no list, no filter, and no division. On the four-million case the recorded averages are about 0.42 microseconds for C against 1.6 microseconds for Python: C wins the constant-factor race by roughly four times, yet both are so far below a millisecond that the result is effectively instant, so the value lies in the technique rather than the language. The Python generator version is arguably the more elegant expression of the idea, while the C version is the more explicit.

  • When a problem asks for a subset of a sequence defined by a regular pattern, derive a direct recurrence on that subset before reaching for a filter - it removes branches and simplifies the code.
  • Exponentially growing sequences have only O(log L) terms below any limit L, so such problems run in logarithmic time regardless of implementation language.
  • Two state variables suffice for any depth-2 linear recurrence; keep memory at O(1) by rolling them forward.
  • A generator is the idiomatic way to express a lazy, open-ended sequence in Python; pairing one with sum() or max() separates generation from aggregation.
  • Parallel assignment updates interdependent variables atomically; in C, use a single explicit temporary to get the same guarantee.
  • Always verify seed values by hand - an off-by-one in initialisation silently corrupts every term a recurrence produces.
