Problem Analysis

Every quadratic irrational - including the square root of any non-square integer - has a continued fraction expansion that is eventually periodic. A continued fraction writes a number as an integer part plus the reciprocal of another such expansion, nested indefinitely; for square roots this nesting settles into a repeating block. My task is to count how many N up to 10,000 have an odd-length repeating block. Perfect squares are excluded outright: their roots are rational, the expansion terminates, and worse, the recurrence below would divide by zero on the first step.

The key observation that makes the problem tractable is that the algorithm for extracting the coefficients - the standard square-root periodic continued fraction recurrence - keeps the entire state in three small integers. At each step the current remainder has the form (sqrt(N) + m) / d, and from (m, d) I read off the next coefficient a. Crucially m is bounded by sqrt(N) and d divides N - m^2 exactly, so the triple (m, d, a) ranges over a finite set. By the pigeonhole principle, the sequence of triples must eventually repeat, and the first repeat closes the cycle: the number of distinct triples seen is the period. Since the period for N up to 10,000 is at most a few hundred, each inner loop runs in roughly O(sqrt(N)) steps, and the overall cost is about O(N_max * sqrt(N_max)) - on the order of 10^6 integer operations, trivially fast.

Solution Approach

Continued-Fraction Period Detection via State-Triple Cycle

Both the Python (p0064_s0.py) and C (p0064_s0.c) files implement the identical algorithm; I describe it once and note the language differences afterward.

Step 1 - Compute a0. For each candidate N I compute a0 = floor(sqrt(N)). If a0^2 == N then N is a perfect square and I skip it. Filtering these at the call site keeps the period routine's precondition - its input is irrational - explicit, and avoids the division-by-zero the recurrence would otherwise hit.

Step 2 - Iterate the recurrence. Starting from m = 0, d = 1, a = a0, I repeatedly apply three integer updates: m = d*a - m, then d = (N - m*m) / d, then a = (a0 + m) / d. It is a provable fact about this periodic continued fraction recurrence that d always divides N - m^2 exactly, so every quantity stays an integer - no floating point appears inside the loop, and there is no rounding error to reason about.

Step 3 - Detect the period. I accumulate the visited triples. When a freshly computed triple already appears in the collection, the cycle has closed, and the period is the count of triples gathered before that repeat. I test against every stored triple rather than only the initial state. In practice the cycle here always begins immediately at the first generated triple, but the general membership check costs nothing in clarity and is the robust, correct cycle-detection condition.

Step 4 - Count odd periods. The outer loop runs over every non-square N from 2 to the limit, calls the period routine, and tallies each odd result.

Language differences. The Python version stores triples in a list of tuples and tests membership with the in operator - O(period) per check, which is negligible because periods are tiny. It uses math.isqrt for an exact integer square root inside the routine, and filters squares with not math.sqrt(n).is_integer(). The C version stores triples in a dynamically allocated array of int[3], starting at capacity 64 and doubling via realloc. Because C's sqrt operates on double, I wrap each call in two corrective while loops to guarantee an exact integer floor. Both produce 1322 for N up to 10,000; the C build runs in about 0.0062 s against 0.0940 s for Python, roughly a 15x speedup.

Programming Techniques

State-Triple Representation of a Discrete Dynamical System

The mathematical object at each step is the quadratic irrational (sqrt(N) + m) / d. Rather than manipulating irrational numbers, I encode the whole state as the comparable integer triple (m, d, a) - a tuple in Python, an int[3] in C. Reducing a state to a single comparable value turns "have I been here before?" into a plain equality test. This "state as a value" pattern is reusable whenever you need to detect cycles in a discrete dynamical system: Collatz-style iterations, finite automata, or visited-board detection in puzzle search.

Cycle Detection by Membership Test

Both solutions detect the period by checking each new state against all previously seen states. Because the period is small, the linear scan per step is negligible. When the state space is small and bounded, a plain list-and-membership cycle detector is simpler than - and just as fast as - more elaborate schemes such as Floyd's tortoise-and-hare, and it directly yields the period length as the size of the accumulated collection.

Defensive Integer Square Root in C

C's sqrt returns a double with only a 53-bit mantissa, so for large N it can land just below the true integer root. After casting, I run one loop that increments the candidate while (a0+1)^2 <= N and another that decrements while a0^2 > N, using long long in the comparisons to avoid overflow. This two-loop correction is the standard defensive idiom whenever you need an exact integer square root in a language lacking a built-in like Python's math.isqrt.

Dynamic Array with Doubling (C)

The visited-triples buffer starts at capacity 64 and doubles via realloc on overflow, giving amortized O(1) appends - each element is copied a constant number of times on average over the buffer's life. I always assign the realloc result to a temporary pointer and check it before overwriting the original, so a failed reallocation does not leak the existing buffer. This is the canonical safe growable-array idiom in C, and 64 is a deliberate starting size: periods here rarely exceed a couple hundred, so growth happens at most once or twice.

Generator Expression Fed to sum (Python)

The Python solve collapses the entire outer loop into a single sum over a generator expression that lazily yields True/False (counted as 1/0) for each qualifying N. It is concise, builds no intermediate list, and reads almost as the problem is stated: count the N with odd period. Whenever you need "how many items satisfy a predicate", a generator expression passed to sum is the idiomatic, memory-efficient Python choice.

Filtering Degenerate Inputs at the Call Site

Skipping perfect squares before entering the period routine serves two purposes: correctness (the recurrence would divide by zero on a square) and clarity (the routine may assume its input is irrational). Placing the guard at the call site rather than inside the core routine keeps both pieces of code clean and makes the precondition explicit - a separation-of-concerns discipline worth applying broadly.

Key Takeaways

The unlocking insight is that the continued fraction expansion of sqrt(N) is fully captured by a small integer triple (m, d, a), that the recurrence on this triple stays entirely in integer arithmetic, and that the bounded state space guarantees an eventually periodic sequence. Once you see this, finding the period reduces to plain cycle detection on a finite state set - a pattern that recurs across number-theory problems.

Both implementations are correct and fast. On the main input (N up to 10,000) the C version runs in about 0.0062 s versus 0.0940 s for Python - roughly a 15x speedup, typical for a tight integer loop where interpreter overhead dominates. For a problem this size Python is more than fast enough and arguably more elegant, thanks to math.isqrt and the generator-and-sum idiom; the C version demonstrates how to reproduce the same algorithm while carefully handling floating-point square roots and manual memory growth.

  • When a process generates a sequence of integer states, represent each state as a single comparable value so cycle detection becomes an equality check; the period ends at the first repeat.
  • For quadratic irrationals, the triple (m, d, a) encoding (sqrt(N)+m)/d with its standard recurrence is a ready-made tool - recognizing it lets you skip rediscovering the math.
  • Keep loops in exact integer arithmetic when the math permits; it eliminates an entire class of floating-point bugs.
  • In C, always wrap sqrt with correction loops (and use wider types in the comparisons) when you need an exact integer square root.
  • Use the doubling-realloc pattern for growable arrays in C, and save the realloc result to a temporary before overwriting the original to stay leak-safe.
  • Filter degenerate inputs at the call site so your core routines can state clean preconditions.
  • In Python, count predicate matches with a generator expression passed to sum - concise and allocation-free.
