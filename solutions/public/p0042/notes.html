Problem Analysis

This problem fuses a simple string-processing task with a number-theoretic membership test. For each word I compute a "word value" by mapping each letter to its 1-based alphabetical position (A=1, ..., Z=26) and summing those positions. I then ask whether that sum is a triangular number - a number of the form t(n) = n(n+1)/2 for some positive integer n, which counts the dots in an equilateral triangular arrangement.

The observation that makes the test cheap is that I never need to build a table of triangular numbers. Because t(n) is a closed-form quadratic, I can invert it: setting n(n+1)/2 = k and solving the quadratic n^2 + n - 2k = 0 with the quadratic formula gives the positive root n = (-1 + sqrt(1 + 8k)) / 2. That n is a whole number exactly when 1 + 8k is a perfect square. So "is k triangular?" reduces to "is 8k+1 a perfect square?" - a constant-time test requiring no precomputation and no chosen upper bound. With roughly two thousand words and each word value bounded well under a few hundred, the work is O(W * L) where W is the word count and L the average word length: linear, and dominated entirely by file I/O and parsing. Both implementations finish in well under a millisecond.

Solution Approach

Direct Triangle Test via Perfect-Square Check (Python and C)

Both files implement the same three-step algorithm; only the language mechanics differ.

Step 1 - load and split. I read the whole file into memory as one string, then split it on commas to recover each quoted word. I chose in-memory splitting over line-by-line reading because the file uses commas, not newlines, as its sole delimiter. In Python this is str.split(","). In C I use strtok_r, the re-entrant variant of strtok, which walks the buffer and writes null terminators in place - cheap because no per-token allocation is needed.

Step 2 - score each word. I strip quote characters and whitespace, then map each remaining letter to its position by subtracting the code point just below 'A'. In Python this is ord(c) - 64 (since ord('A') is 65), wrapped in a generator expression fed to sum. In C it is *c - 'A' + 1 inside an explicit loop; the C version also handles lowercase defensively with a parallel *c - 'a' + 1 branch. The Python scoring function carries an lru_cache decorator, which memoises results - harmless here and a sensible default for a pure function called in a loop.

Step 3 - triangle test. I check whether 8*n + 1 is a perfect square. In Python I compute (8*n + 1) ** 0.5 and call float.is_integer() on it; this is safe because all word values are small enough for the double-precision square root to be exact. In C I take sqrt, round to the nearest integer with round, then square that integer back and compare against 8*n + 1 using 64-bit integer arithmetic. That round-trip guards against any floating-point rounding error.

The final answer is the count of words passing the test - tallied via sum over a boolean generator in Python, and via an explicit counter in C.

Programming Techniques

Closed-Form Membership Test for a Quadratic Sequence

Rather than generating triangular numbers up to a bound and storing them in a set, I invert the defining formula to get a direct predicate. Whenever a sequence has a closed form, it is worth solving for the index and asking when that index is an integer. Here "is k triangular?" becomes "is 8k+1 a perfect square?". The same trick yields tests for pentagonal, hexagonal, and other polygonal numbers. Reaching for the algebraic inverse first avoids both memory use and the awkward question of how high to generate.

Perfect-Square Detection Using an Integer Round-Trip

Computing sqrt and checking integrality is the natural idea, but floating-point roots can be off by a hair. The C solution rounds the root to the nearest integer and squares it back to compare exactly against the original. This "compute approximate, round, verify exactly" pattern is a standard defensive idiom for using a floating-point intermediate to answer an integer question. Python's float.is_integer() is the equivalent test; for the small values here both are safe.

Generator Expressions and sum() for Counting

The Python idiom sum(predicate(x) for x in iterable) counts matching elements by exploiting that True coerces to 1 and False to 0. It avoids a mutable counter, states intent directly, and stays lazy - no intermediate list is built. I reach for this whenever I need a conditional count over a sequence.

Memoization with lru_cache

Decorating the scoring function with lru_cache caches its results - a form of memoization. The payoff here is small since word repetition is unlikely, but caching pure, deterministic functions called in tight loops is a low-cost habit that pays off whenever the input domain turns out smaller than expected.

strtok_r for In-Place Tokenisation in C

C's strtok keeps hidden global state and is therefore not re-entrant. I use strtok_r, the POSIX variant that stores its position in a caller-supplied pointer, so it is safe under nesting or threading. It mutates the buffer by writing null terminators, which is fine here because the buffer is a writable heap copy returned by get_text_file. Preferring the re-entrant form costs nothing and avoids a class of subtle bugs.

Key Takeaways

The unlocking insight is to test triangular-number membership algebraically: k is triangular if and only if 8k+1 is a perfect square. That replaces any lookup table with an O(1) check and removes the need to pick a bound. Both solutions share this identical algorithm; the runtime difference is purely the cost of language abstraction. The C build runs in about 0.0001 s versus 0.0004 s for Python - roughly four times faster - reflecting interpreter and object-model overhead rather than any algorithmic gap, since the work is tiny and linear either way.

  • When a sequence has a closed-form formula, invert it to get a direct membership predicate - no table, no upper bound to guess.
  • To answer an integer question through floating-point, round the result and verify by exact integer arithmetic to defend against rounding error.
  • Use sum(predicate(x) for x in iterable) for clean, allocation-free conditional counting in Python.
  • Prefer the re-entrant strtok_r over strtok in C; it removes hidden global state at no cost.
  • Memoize pure functions called in loops by default - the habit is cheap and occasionally a large win.
