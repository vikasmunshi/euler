Problem 64: Odd Period Square Roots [Level 3]

Problem Statement

All square roots are periodic when written as continued fractions and can be written in the form:

$$\sqrt{N}=a_0 + \dfrac 1 {a_1 + \dfrac 1 {a_2 + \dfrac 1 {a_3 + \dots}}}$$

For example, let us consider $\sqrt{23}:$

$$\sqrt{23} = 4 + \sqrt{23}-4=4 + \dfrac 1 {\dfrac 1 {\sqrt{23}-4}} = 4+\dfrac 1 {1 + \dfrac{\sqrt{23}-3}7}$$

If we continue we would get the following expansion:

$$\sqrt{23}=4 + \dfrac 1 {1 + \dfrac 1 {3+ \dfrac 1 {1 + \dfrac 1 {8+ \dots}}}}$$

The process can be summarised as follows:

$$\begin{align} \quad \quad a_0 &= 4, \frac 1 {\sqrt{23}-4}=\frac {\sqrt{23}+4} 7=1+\frac {\sqrt{23}-3} 7 \\ \quad \quad a_1 &= 1, \frac 7 {\sqrt{23}-3}=\frac {7(\sqrt{23}+3)} {14}=3+\frac {\sqrt{23}-3} 2 \\ \quad \quad a_2 &= 3, \frac 2 {\sqrt{23}-3}=\frac {2(\sqrt{23}+3)} {14}=1+\frac {\sqrt{23}-4} 7 \\ \quad \quad a_3 &= 1, \frac 7 {\sqrt{23}-4}=\frac {7(\sqrt{23}+4)} 7=8+\sqrt{23}-4 \\ \quad \quad a_4 &= 8, \frac 1 {\sqrt{23}-4}=\frac {\sqrt{23}+4} 7=1+\frac {\sqrt{23}-3} 7 \\ \quad \quad a_5 &= 1, \frac 7 {\sqrt{23}-3}=\frac {7 (\sqrt{23}+3)} {14}=3+\frac {\sqrt{23}-3} 2 \\ \quad \quad a_6 &= 3, \frac 2 {\sqrt{23}-3}=\frac {2(\sqrt{23}+3)} {14}=1+\frac {\sqrt{23}-4} 7 \\ \quad \quad a_7 &= 1, \frac 7 {\sqrt{23}-4}=\frac {7(\sqrt{23}+4)} {7}=8+\sqrt{23}-4 \end{align}$$

It can be seen that the sequence is repeating. For conciseness, we use the notation $\sqrt{23}=[4;(1,3,1,8)]$, to indicate that the block (1,3,1,8) repeats indefinitely.

The first ten continued fraction representations of (irrational) square roots are:

$\quad \quad \sqrt{2}=[1;(2)]$, period=$1$
$\quad \quad \sqrt{3}=[1;(1,2)]$, period=$2$
$\quad \quad \sqrt{5}=[2;(4)]$, period=$1$
$\quad \quad \sqrt{6}=[2;(2,4)]$, period=$2$
$\quad \quad \sqrt{7}=[2;(1,1,1,4)]$, period=$4$
$\quad \quad \sqrt{8}=[2;(1,4)]$, period=$2$
$\quad \quad \sqrt{10}=[3;(6)]$, period=$1$
$\quad \quad \sqrt{11}=[3;(3,6)]$, period=$2$
$\quad \quad \sqrt{12}=[3;(2,6)]$, period=$2$
$\quad \quad \sqrt{13}=[3;(1,1,1,1,6)]$, period=$5$

Exactly four continued fractions, for $N \le 13$, have an odd period.

How many continued fractions for $N \le 10\,000$ have an odd period?

  • main: solve(max_limit=10000) → 1322

Mathematical Complexity

This problem sits in the theory of continued fractions for quadratic irrationals. Every irrational square root sqrt(N) has a periodic continued fraction [a0; (a1, a2, ..., ak)] where the repeating block always ends with the term 2*a0. The standard algorithm for computing the period tracks a triple (m, d, a) representing the residual (sqrt(N) + m) / d at each step, updated via the recurrence m = d*a - m, d = (N - m^2) / d, a = floor((a0 + m) / d). Because both m and d are bounded by O(sqrt(N)), the triple must eventually repeat, and the period length is at most O(sqrt(N)). Accumulating this work over all non-square N up to a limit L gives total complexity of roughly O(L * sqrt(L)), placing the solution in the O(N^(3/2)) complexity class.

A deeper number-theoretic connection is worth noting: the parity of the continued fraction period determines solvability of the negative Pell equation x^2 - N*y^2 = -1. This equation has integer solutions if and only if the continued fraction period of sqrt(N) is odd. Counting odd-period square roots is therefore directly equivalent to counting those N for which the negative Pell equation is solvable - the problem is more deeply rooted in Diophantine equation theory than it first appears, and this connection resurfaces in Problem 66.

Solution Approach

State-Tracking Continued Fraction Period Detection (Python and C)

Both p0064_s0.py and p0064_s0.c implement the same standard textbook recurrence. I seed a0 = floor(sqrt(N)), then iteratively apply m = d*a - m, d = (N - m^2) / d, a = floor((a0 + m) / d), accumulating visited (m, d, a) triples until a triple appears for the second time. The count of distinct triples accumulated before the repeat is exactly the period length. I then count all non-square N in [2, max_limit] for which that period is odd.

In the Python version I use math.isqrt to seed a0 and filter perfect squares with math.sqrt(n).is_integer(). The visited triples are stored in a list p and membership is tested with the in operator, making cycle detection O(period^2) per N rather than O(period). For N up to 10,000 the periods are short enough that this is not a bottleneck in practice, but replacing the list with a set would be a cleaner and asymptotically better choice.

The C version mirrors the algorithm exactly. I maintain a dynamically allocated array of (m, d, a) triples, doubling its capacity as needed via realloc. The inner linear scan for duplicates carries the same O(period^2) per-N characteristic as the Python list approach. I use an integer square root correction loop rather than relying on floating-point sqrt alone, which avoids rounding errors near large perfect squares. Both solutions produce the correct answer of 1322 for N up to 10,000.

Learnings

The key insight for me was that the entire period computation is pure integer arithmetic once the initial triple is established. I only need one approximate square root call to seed a0; everything after that is exact integer division that never accumulates floating-point error. The reason d always divides (N - m^2) exactly is that the algebra of rationalising the denominator at each step guarantees integer quotients - once I convinced myself of this, the algorithm felt completely trustworthy.

The C solution runs the main case in about 0.0077 seconds versus 0.139 seconds for Python, a roughly 18x speedup. This is expected given C's tight arithmetic loop and manual memory management versus Python's interpreted overhead and object allocation for tuples. Both are fast enough for the given limit, but the C version would scale considerably better if the limit were raised significantly.

  • The triple (m, d, a) is a complete state descriptor for the continued fraction iteration; once it repeats, the period is exactly the number of distinct triples accumulated before that repetition.
  • Replacing the list membership check with a set would reduce cycle detection from O(period^2) to O(period) per N - a worthwhile cleanup, especially at larger limits.
  • Using math.sqrt(n).is_integer() for perfect-square filtering is readable but math.isqrt(n)**2 == n is more numerically robust and worth preferring for large N.
  • In C, correcting the integer square root with a small adjustment loop after sqrt() is a reliable pattern for avoiding off-by-one errors from floating-point rounding near perfect squares.
  • The connection between odd continued fraction period and solvability of x^2 - N*y^2 = -1 is a key fact to keep in mind for Problem 66 and related Pell equation problems.
