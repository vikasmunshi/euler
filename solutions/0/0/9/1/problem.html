Problem 91: Right Triangles with Integer Coordinates [Level 6]

Problem Statement

The points $P(x_1, y_1)$ and $Q(x_2, y_2)$ are plotted at integer co-ordinates and are joined to the origin, $O(0,0)$, to form $\triangle OPQ$.


There are exactly fourteen triangles containing a right angle that can be formed when each co-ordinate lies between $0$ and $2$ inclusive; that is, $0 \le x_1, y_1, x_2, y_2 \le 2$.


Given that $0 \le x_1, y_1, x_2, y_2 \le 50$, how many right triangles can be formed?

  • main: solve(coordinate_limit=50) → 14234

Mathematical Complexity

The problem requires counting every right triangle OPQ whose vertices are lattice points within an N by N grid. The key mathematical insight is that the three vertices compete to hold the right angle, and each class behaves very differently. Triangles where the right angle sits on a coordinate axis - at O itself, at a point on the positive x-axis, or at a point on the positive y-axis - reduce to a closed-form count. For a point P on the positive x-axis there is exactly one column-mate Q directly above it that makes the angle at P a right angle, giving N such triangles per axis; the origin contributes N squared axis-aligned right angles from pairing any x-axis point with any y-axis point. The three axis classes together contribute exactly 3N squared triangles.

The subtler class is triangles whose right angle sits at a strictly interior lattice point P = (x, y) with x and y both positive. The segment OP has direction (x, y), so the perpendicular direction is (-y, x). In its reduced primitive form this step is (-y/g, x/g) where g = gcd(x, y). A valid Q must lie within the grid, so the number of usable steps in the positive perpendicular direction is bounded by two independent floor-division constraints: how many steps fit before the x-coordinate drops to zero, and how many fit before the y-coordinate exceeds N. The exact per-point count is the minimum of those two quantities. Summing over all O(N squared) interior candidates, each requiring one GCD computation, gives an overall asymptotic complexity of O(N squared log N). The label symmetry between P and Q - swapping them gives a congruent triangle with the right angle at the other non-origin vertex - allows the enumeration to cover only y in [1, N-1] and multiply the interior count by two at the end, avoiding any double-counting.

Solution Approach

GCD-Based Perpendicular Ray Enumeration (s0 - Python and C)

Both implementations express the answer as the sum of two independent parts. The first part is computed directly as triangles_at_origin = 3 * N * N, covering all right angles that touch either coordinate axis. The second part iterates over every interior base point P = (x, y) with x in [1, N] and y in [1, N-1], computes m = gcd(x, y), and evaluates the two boundary bounds: x * m // y for the lower-x boundary and m * (N - y) // x for the upper-y boundary. The valid Q count for that P is the minimum of the two bounds, and the total interior sum is multiplied by two to account for the symmetric Q-right-angle class.

The C version defines a small static Euclidean gcd helper and uses explicit nested loops with integer arithmetic. The Python version achieves the same logic in a single generator expression; the idiom for m in [math.gcd(x, y)] inside the comprehension binds the GCD once per pair without needing a helper function or a walrus operator. The arithmetic is otherwise line-for-line equivalent between the two languages. Both produce 14 for the development input and 14234 for the main input. The C build is effectively instantaneous while the Python version completes in roughly 0.4 ms - a real constant-factor difference, but both are far faster than the problem requires.

Learnings

The insight that unlocked the solution was recognising that the right angle's location completely determines which geometric argument applies. Once I saw that axis-touching cases collapse to the closed-form 3N squared and that interior cases become a pure lattice-direction question, the path forward was clear. The second critical insight was that the primitive perpendicular lattice step at (x, y) is controlled by gcd(x, y), so counting in-grid points along that ray reduces to two independent floor-division bounds - one per relevant grid boundary - whose minimum gives the exact answer.

  • Decomposing a geometry counting problem by the location of the special vertex - here the right angle - can turn an otherwise intractable enumeration into simple closed forms per class.
  • The primitive lattice step perpendicular to a vector (x, y) is (-y/g, x/g) where g = gcd(x, y); this GCD reduction appears whenever perpendicular directions need to be discretised on an integer grid.
  • Boundary constraints on a lattice ray translate directly into floor-division bounds, one per relevant grid edge, and the minimum of those bounds is the exact in-grid lattice-point count along the ray.
  • Exploiting label symmetry to halve an enumeration and multiply at the end is cleaner than iterating explicitly over both orientations and risk double-counting.
  • The Python idiom for m in [expr] inside a generator expression is a concise way to bind a computed subexpression once without introducing a nested function or a walrus operator.
