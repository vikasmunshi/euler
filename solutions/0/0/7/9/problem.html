Problem 79: Passcode Derivation [Level 2]

Problem Statement

A common security method used for online banking is to ask the user for three random characters from a passcode. For example, if the passcode was 531278, they may ask for the 2nd, 3rd, and 5th characters; the expected reply would be: 317.

The text file, keylog.txt , contains fifty successful login attempts.

Given that the three characters are always asked for in order, analyse the file so as to determine the shortest possible secret passcode of unknown length.

Mathematical Complexity

Each three-character login attempt encodes strict ordering constraints: if digit A appears before digit B in an attempt, then A must precede B somewhere in the full passcode. These pairwise constraints are exactly the edges of a directed acyclic graph (DAG) whose nodes are the distinct digit characters seen across all attempts. Recovering the shortest passcode is therefore a topological ordering problem - find the unique linear extension of the partial order defined by the union of all observed constraints. The problem guarantees a unique shortest solution, which means the induced DAG must have a unique topological ordering, implying the partial order is in fact a total order.

Let D be the number of distinct digit characters and L the number of login attempts, with k characters per attempt (k = 3 here). Building the constraint graph takes O(L * (k - 1)) time. The topological sort I use repeatedly scans all remaining nodes to find one with an empty active-successor set, giving O(D^2) for that phase. Since D is at most 10 and L is 50 for this input, the entire computation is effectively constant-time regardless of implementation language.

Solution Approach

Topological Sort via Reverse Leaf Removal (Successor-Graph Method) - Python and C

I implemented the same algorithm in both Python ( p0079_s0.py) and C ( p0079_s0.c) and cover them together here. My approach was to build a successor graph - a mapping from each digit character to the set of characters that must appear after it in the passcode. For every deduplicated login attempt I added a directed edge from the character at position i to the character at position i+1, for i from 0 to k-2. I then performed a reverse topological sort: repeatedly locate the node whose active successor set is empty (meaning no remaining character is constrained to follow it), prepend that character to the front of the growing passcode string, and remove it from the graph along with all references to it in other nodes' successor sets. This builds the passcode from right to left. If at any iteration no such node exists, a cycle is present, which the problem guarantees cannot happen for valid input.

In Python I used a dict[str, set[str]] for the successor graph. Deduplication of login attempts was free via a set applied to the split lines. The for-else construct in the sort loop lets me cleanly raise a ValueError if no tail node is found. In C I represented the same structure with a fixed-size bitmask array ( unsigned int successor[MAX_DIGITS]) alongside a parallel active[] flag array. When testing whether a node's successor set is empty, I mask out bits corresponding to inactive nodes before comparing against zero. The C version avoids heap allocation for the graph entirely and handles file I/O with manual path construction relative to the executable directory. Both versions produce the answer 73162890.

Learnings

The key insight was recognising that partial character-ordering observations from login attempts map directly onto directed graph edges, and that reconstructing the full passcode is simply a topological sort problem. Once I framed it that way, the implementation was immediate. I also noted that removing nodes from the tail (no successors) and prepending them builds the result in reverse - this is a valid alternative direction to the more commonly taught Kahn's algorithm, which removes nodes from the head (no predecessors) and appends.

The C solution runs in effectively zero measured time (0.0000 s) while the Python solution takes about 0.2 ms - overwhelmingly interpreter and import overhead rather than any algorithmic cost given the tiny input size. The bitmask representation of successor sets in C is a compact and elegant way to handle small fixed-size node sets without any heap allocation. For larger graphs, an adjacency list or bitset library would be more appropriate, but here a plain unsigned int field per node is entirely sufficient.

  • Partial ordering constraints from sequential observations map directly to directed graph edges - recognising this framing immediately unlocks the solution.
  • Topological sort with leaf removal from the tail (no successors) builds the result in reverse and is an equally correct alternative to Kahn's predecessor-based algorithm.
  • Deduplicating input with a set before building the constraint graph avoids redundant edges and keeps the structure minimal.
  • A bitmask representation of successor sets in C is compact and requires no heap allocation when the number of nodes is small and statically bounded.
  • The problem's guarantee of a unique shortest passcode implies the induced DAG has a unique topological ordering - verifying this property before implementation confirms a single definitive answer will always be produced.
