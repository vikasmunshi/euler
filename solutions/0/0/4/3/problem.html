Problem 43: Sub-string Divisibility [Level 1]

Problem Statement

The number, $1406357289$, is a $0$ to $9$ pandigital number because it is made up of each of the digits $0$ to $9$ in some order, but it also has a rather interesting sub-string divisibility property.

Let $d_1$ be the $1$ st digit, $d_2$ be the $2$ nd digit, and so on. In this way, we note the following:

  • $d_2d_3d_4=406$ is divisible by $2$
  • $d_3d_4d_5=063$ is divisible by $3$
  • $d_4d_5d_6=635$ is divisible by $5$
  • $d_5d_6d_7=357$ is divisible by $7$
  • $d_6d_7d_8=572$ is divisible by $11$
  • $d_7d_8d_9=728$ is divisible by $13$
  • $d_8d_9d_{10}=289$ is divisible by $17$

Find the sum of all $0$ to $9$ pandigital numbers with this property.

Mathematical Complexity

The naive search space is all 10! = 3,628,800 permutations of the digits 0-9. The problem imposes seven overlapping divisibility constraints on consecutive three-digit windows, using the primes 2, 3, 5, 7, 11, 13, and 17 in order. Each window shares two digits with its immediate neighbours, so the constraints form a tightly coupled chain rather than seven independent conditions. The valid set turns out to be only six pandigital numbers, illustrating how drastically the chain structure prunes the search. The asymptotic complexity of the constraint-chain approach is far below O(10!): starting from 44 three-digit multiples of 17 and pruning one digit at a time, the branching factor at each recursive level is at most 7 (unused digits), and the divisibility check eliminates almost all branches immediately.

This is a constraint satisfaction problem with a natural linear-chain topology. Fixing the rightmost three-digit window and extending the partial number one digit to the left at each step means every new divisibility condition can be checked the moment the relevant digit is placed, before any further recursion. The total work is bounded by the product of the surviving branching factors across all levels, which is tiny in practice - empirically, only six complete pandigitals are ever assembled.

Solution Approach

Right-to-Left Recursive Construction with Constraint Pruning (s0 - Python and C)

I pre-enumerated all 44 three-digit multiples of 17 that use distinct digits and hard-coded them as the seed set representing positions d8d9d10. The recursive generator then works backwards through the divisor chain - 13, 11, 7, 5, 3, 2 - prepending one new digit at each level. At every extension, I check two things: that the new digit has not already appeared in the partial number, and that the leftmost three characters of the extended string are divisible by the current divisor in the chain. Branches that fail either check are cut immediately without further recursion. When the partial string reaches nine digits (covering positions d2 through d10), the single remaining unused digit is prepended as d1, completing a valid pandigital candidate whose integer value is added to the running total.

The Python and C implementations follow exactly the same algorithm. In Python, I encoded the backward divisor chain as a dictionary {3: 2, 5: 3, 7: 5, 11: 7, 13: 11, 17: 13} and represented partial numbers as strings, making digit membership testing and prefix extraction natural. The generator accepts an optional --show flag that prints each valid number individually. In C, I looped over the 44 seeds at the top level and used a recursive helper for the backward extension, encoding the divisor chain in a next_divisor switch statement and accumulating the sum in a global variable. Both versions produce the correct answer of 16695334890.

Learnings

The critical insight was recognising that the overlapping windows create a dependency chain best exploited by building the number from right to left rather than generating all pandigital permutations and filtering. Starting from the tightest single constraint - divisibility by 17, the largest prime - seeds a small set of 44 candidate strings. Each backward step could multiply the candidate count by at most 7, but the divisibility check prunes almost everything immediately. Only six valid pandigitals survive the full construction, which confirms how powerful early pruning is when constraints overlap in a chain.

A technique worth remembering for similar problems is constraint chaining with shared overlap: whenever consecutive constraints share sub-structure, fixing one end and extending incrementally while checking each new constraint on arrival is far more efficient than generating candidates up front and post-filtering. Representing partial numbers as strings rather than integers made both digit membership testing and three-character window extraction readable and straightforward in both languages. Pre-computing the valid multiples of 17 as a literal list avoids repeated modulo arithmetic in the innermost loop and makes the seed set immediately visible.

Only one algorithm was implemented, so there is no meaningful comparison between approaches. The benchmark results do show the expected language gap: the C solution runs in effectively zero measured time at the microsecond scale, while the Python solution completes in roughly 0.2 ms. Both are orders of magnitude faster than a brute-force permutation filter would be, and both are instantaneous in practice for a problem this tightly constrained.
