Problem 45: Triangular, Pentagonal, and Hexagonal [Level 0]

Problem Statement

Triangle, pentagonal, and hexagonal numbers are generated by the following formulae:

Triangle $T_n=n(n+1)/2$ $1, 3, 6, 10, 15, \dots$
Pentagonal $P_n=n(3n - 1)/2$ $1, 5, 12, 22, 35, \dots$
Hexagonal $H_n=n(2n - 1)$ $1, 6, 15, 28, 45, \dots$

It can be verified that $T_{285} = P_{165} = H_{143} = 40755$.

Find the next triangle number that is also pentagonal and hexagonal.

Mathematical Complexity

The core mathematical idea is inverting the quadratic generating formulae for polygonal numbers to produce O(1) membership tests. Given a candidate value x, I can check whether it is pentagonal by solving 3n^2 - n - 2x = 0 for n, yielding n = (1 + sqrt(1 + 24x)) / 6. The value is pentagonal if and only if 1 + 24x is a perfect square whose root is congruent to 5 mod 6, ensuring the numerator divides evenly by 6 and the resulting index is a positive integer. The hexagonal check follows the same pattern: solving 2n^2 - n - x = 0 gives n = (1 + sqrt(1 + 8x)) / 4, which is a positive integer exactly when 1 + 8x is a perfect square whose root is congruent to 3 mod 4. A useful structural shortcut is that every hexagonal number H_n = n(2n - 1) is equal to the triangular number T_{2n-1}, so all hexagonal numbers are already triangular. This means I only need to check two independent conditions - pentagonality and hexagonality - rather than three.

Each candidate requires only O(1) arithmetic: two square root evaluations and two modular residue checks. The overall search is linear in the triangular index of the answer, giving O(N) complexity where N is that index. For this problem N is small enough that both implementations finish in well under a second.

Solution Approach

Linear Scan with Inverse Quadratic Membership Tests (s0 - Python and C)

Both the Python and C implementations share the same algorithm. I iterate over triangular indices starting from the known example at n = 285, computing each triangular number t = n * (n + 1) / 2 and applying the two floating-point inverse-formula checks. The pentagonal test evaluates sqrt(1 + 24 * t) % 6 == 5.0 and the hexagonal test evaluates sqrt(1 + 8 * t) % 4 == 3.0. A non-integer square root will not satisfy the modular equality, so a single floating-point evaluation simultaneously confirms both that the discriminant is a perfect square and that the resulting index is a positive integer - no separate integer cast or rounding step is needed. I return the first triangular number beyond the known example that passes both tests, which produces the answer 1533776805.

The Python version uses the walrus operator ( while n := (n + 1):) to advance the index compactly in-place. The C version uses a plain while (1) loop with n++ and fmod for the modular checks, reading the starting index from argv. Both produce the correct answer. The C version runs in about 0.0005 s compared to 0.0130 s for Python - roughly a 26x speedup - which reflects the usual interpreter overhead rather than any algorithmic difference.

Learnings

The key insight is that inverting a polygonal formula turns a set-membership search into an O(1) arithmetic test. Instead of generating all pentagonal and hexagonal numbers up to some bound and intersecting three sets, I derived closed-form inverse expressions and reduced the problem to a single linear scan. The modular residue condition ( % 6 == 5.0 and % 4 == 3.0) encodes two requirements at once: integrality of the square root and divisibility of the numerator by the denominator. Relying on floating-point modular arithmetic as a combined integrality-and-residue test is convenient here because the numbers involved are small enough that double precision is exact throughout the search.

The structural observation that all hexagonal numbers are a subset of triangular numbers is worth keeping in mind. It reduces three membership conditions to two and would also support an even more direct alternative: iterate over hexagonal indices and check only pentagonality, halving the number of candidates examined. For future problems involving any k-gonal number type, the general pattern is: derive the quadratic inverse, identify the discriminant, determine the required residue class of its integer square root, and use a floating-point modular check as a combined integrality-and-residue test - avoiding large set construction entirely. Since only one algorithmic approach was implemented here, there is no comparison between strategies, but the technique generalises cleanly across all polygonal number problems.
