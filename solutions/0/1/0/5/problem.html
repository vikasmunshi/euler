Problem 105: Special Subset Sums [Level 11]

Problem Statement

Let $S(A)$ represent the sum of elements in set $A$ of size $n$. We shall call it a special sum set if for any two non-empty disjoint subsets, $B$ and $C$, the following properties are true:

  1. $S(B) \ne S(C)$; that is, sums of subsets cannot be equal.
  2. If $B$ contains more elements than $C$ then $S(B) \gt S(C)$.

For example, $\{81, 88, 75, 42, 87, 84, 86, 65\}$ is not a special sum set because $65 + 87 + 88 = 75 + 81 + 84$, whereas $\{157, 150, 164, 119, 79, 159, 161, 139, 158\}$ satisfies both rules for all possible subset pair combinations and $S(A) = 1286$.

Using sets.txt (right click and "Save Link/Target As..."), a 4K text file with one-hundred sets containing seven to twelve elements (the two examples given above are the first two sets in the file), identify all the special sum sets, $A_1, A_2, \dots, A_k$, and find the value of $S(A_1) + S(A_2) + \cdots + S(A_k)$.

NOTE: This problem is related to Problem 103 and Problem 106 .

Mathematical Complexity

This problem sits at the intersection of combinatorics and set theory. Given a set of n elements, I need to verify two properties over all pairs of non-empty disjoint subsets: no two such subsets may share the same sum (rule 1), and if one subset is strictly larger than another, its sum must also be strictly larger (rule 2). The number of non-empty subsets of an n -element set is 2^n - 1, so the number of ordered disjoint pairs is bounded by roughly 3^n (since each element can belong to subset B, subset C, or neither). For the sizes in the input file (7 to 12 elements), 3^12 = 531441, which is entirely tractable.

The key algorithmic idea is to precompute all subset sums using a bitmask dynamic-programming recurrence: for a mask with lowest set bit lsb, the sum is subset_sum[mask ^ lsb] + elements[i]. This gives all 2^n subset sums in O(2^n) time and space. Checking disjoint pairs is then a matter of iterating over all masks b_mask and, for each, iterating over all submasks of its bitwise complement using the standard submask enumeration trick. The overall check is O(3^n) per set.

Solution Approach

Bitmask Subset Enumeration (Algorithm 0 - Python and C)

I implemented the same algorithm in both Python ( p0105_s0.py) and C ( p0105_s0.c). The approach is identical in both:

  • Read the input file line by line, parsing each comma-separated line into a list of integers.
  • Precompute subset_sum[mask] for every non-empty bitmask using the LSB recurrence.
  • Precompute subset_size[mask] (popcount) for every mask.
  • For each candidate subset B (every non-empty mask b_mask), compute its complement and enumerate all non-empty submasks C of that complement using the trick c_mask = (c_mask - 1) & complement.
  • For each disjoint pair (B, C), check rule 1 (sums differ) and rule 2 (if |B| > |C| then S(B) > S(C)). If either rule fails, the set is not special.
  • Sum the elements of every set that passes both checks.

In the C version I used __builtin_ctz to find the index of the lowest set bit, which is slightly faster than the Python equivalent of lsb.bit_length() - 1. The C version also dynamically allocates the subset sum and size arrays per set and frees them afterwards. Both versions short-circuit as soon as a violation is found, avoiding unnecessary work on non-special sets.

Learnings

The key insight was recognising that the submask enumeration idiom - iterating over all subsets of a given mask using c_mask = (c_mask - 1) & complement - directly gives me exactly the disjoint pairs I need. By fixing B and iterating over all submasks of the complement of B, I naturally cover every pair of non-empty disjoint subsets exactly once per ordered direction, which is sufficient because the two rules are both directional or symmetric enough to catch all violations.

The benchmark results highlight the expected language gap: the C solution runs in about 0.0045 s versus 0.295 s for Python - roughly a 65x speedup - even though both implement the same algorithm. For a problem of this size Python is perfectly adequate, but the C translation is a useful exercise. The precomputed subset-sum table is the most important optimisation; recomputing sums on the fly inside nested loops would be several times slower. The submask enumeration trick and the DP recurrence for subset sums are both techniques worth remembering for any future problem that requires reasoning over all subsets or pairs of disjoint subsets.
