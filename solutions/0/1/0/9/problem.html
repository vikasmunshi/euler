Problem 109: Darts [Level 11]

Problem Statement

In the game of darts a player throws three darts at a target board which is split into twenty equal sized sections numbered one to twenty.


The score of a dart is determined by the number of the region that the dart lands in. A dart landing outside the red/green outer ring scores zero. The black and cream regions inside this ring represent single scores. However, the red/green outer ring and middle ring score double and treble scores respectively.

At the centre of the board are two concentric circles called the bull region, or bulls-eye. The outer bull is worth 25 points and the inner bull is a double, worth 50 points.

There are many variations of rules but in the most popular game the players will begin with a score 301 or 501 and the first player to reduce their running total to zero is a winner. However, it is normal to play a "doubles out" system, which means that the player must land a double (including the double bulls-eye at the centre of the board) on their final dart to win; any other dart that would reduce their running total to one or lower means the score for that set of three darts is "bust".

When a player is able to finish on their current score it is called a "checkout" and the highest checkout is 170: T20 T20 D25 (two treble 20s and double bull).

There are exactly eleven distinct ways to checkout on a score of 6:

D3
D1 D2
S2 D2
D2 D1
S4 D1
S1 S1 D2
S1 T1 D1
S1 S3 D1
D1 D1 D1
D1 S2 D1
S2 S2 D1

Note that D1 D2 is considered different to D2 D1 as they finish on different doubles. However, the combination S1 T1 D1 is considered the same as T1 S1 D1.

In addition we shall not include misses in considering combinations; for example, D3 is the same as 0 D3 and 0 0 D3.

Incredibly there are 42336 distinct ways of checking out in total.

How many distinct ways can a player checkout with a score less than 100?

Mathematical Complexity

This problem is fundamentally a combinatorics counting exercise. The key insight is that a checkout consists of at most three darts, with the last dart always being a double. The first two darts (when used) can be any of the 62 scoring options - 21 singles (S1-S20 plus S25), 21 doubles (D1-D20 plus D50), and 20 trebles (T1-T20; there is no treble bull). The ordering rule is subtle: the final double is fully ordered (D1 followed by D2 is distinct from D2 followed by D1), but the pair of non-final darts is unordered (S1 T1 is the same combination as T1 S1). This means I need ordered selection for the finishing double but unordered (with repetition) selection for the pre-darts.

The search space is tiny. For 3-dart checkouts the number of unordered pairs with repetition from 62 elements is C(62+1, 2) = 1953, and for each pair I check 21 possible finishing doubles, giving under 41000 combinations to test. The total complexity is O(n^2 * m) where n=62 is the number of non-final dart options and m=21 is the number of doubles. This is effectively O(1) in practice for any reasonable limit.

Solution Approach

Direct Enumeration (s0 - Python and C)

I split the problem into three cases based on how many darts are thrown, then enumerate directly.

  • 1-dart checkout: Just a finishing double. I count all doubles strictly less than max_limit. There are at most 21 to check.
  • 2-dart checkout: One pre-dart (any of the 62 options) followed by a finishing double. The pre-dart is fully ordered with respect to the double, so I iterate over all 62 pre-darts and all 21 doubles and count pairs whose sum is less than max_limit.
  • 3-dart checkout: An unordered pair of pre-darts followed by a finishing double. I use a double loop with j >= i (indices into the pre-darts array) to enumerate unordered pairs with repetition, compute their sum, then count valid finishing doubles. This correctly handles the rule that S1 T1 D1 and T1 S1 D1 are the same checkout.

The Python and C implementations follow the same algorithm exactly. The pre-darts array is constructed identically in both: singles (21 elements), doubles (21 elements), trebles (20 elements). The C version uses fixed-size arrays and nested loops with atoi for argument parsing, while the Python version uses list comprehensions and a generator expression for the innermost count. Both accept max_limit as a command-line argument, making them testable for any target score, not just 100.

Learnings

The key "aha moment" for me was carefully reading the ordering rules. The problem states that D1 D2 differs from D2 D1 (because they finish on different doubles), but S1 T1 D1 equals T1 S1 D1 (the non-final darts are unordered). This means only the finishing double carries positional identity - the pre-darts before it do not. Once I understood that, the implementation was straightforward: iterate the finishing double in the outermost distinguishing role, and enumerate the pre-darts as an unordered multiset using index-based lower triangular iteration.

Another point worth remembering: the problem says to exclude misses, so a 1-dart checkout (just a double) counts as a valid single entry, not three distinct entries corresponding to 0 0 D, 0 D, and D. This simplifies the counting considerably.

I only implemented one algorithm here, so there is no meaningful comparison between approaches. The benchmark results confirm that both the Python and C versions are essentially instantaneous - Python finishes in about 2.2 ms and C in under 0.1 ms. The problem is so small that algorithmic improvements would be academic. If I were to extend this to much larger limits the same structure would still work; the only change would be replacing the max_limit threshold check with a tighter bound. The main technique worth remembering for future problems is the unordered-with-repetition enumeration pattern using a double loop with j >= i, which avoids double-counting symmetric pairs without needing an explicit set or deduplication step.
