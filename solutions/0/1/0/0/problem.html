Problem 100: Arranged Probability [Level 7]

Problem Statement

If a box contains twenty-one coloured discs, composed of fifteen blue discs and six red discs, and two discs were taken at random, it can be seen that the probability of taking two blue discs, $P(\text{BB}) = (15/21) \times (14/20) = 1/2$.

The next such arrangement, for which there is exactly $50\%$ chance of taking two blue discs at random, is a box containing eighty-five blue discs and thirty-five red discs.

By finding the first arrangement to contain over $10^{12} = 1\,000\,000\,000\,000$ discs in total, determine the number of blue discs that the box would contain.

Test Cases

  • dev: solve(total_discs=120) → 85
  • main: solve(total_discs=1000000000000) → 756872327473

Mathematical Complexity

The probability condition P(BB) = 1/2 with b blue discs out of n total translates to b(b-1) / (n(n-1)) = 1/2, or equivalently 2b(b-1) = n(n-1). This is a quadratic Diophantine equation in two unknowns. The substitution x = 2n - 1 and y = 2b - 1 eliminates the linear terms and centres both variables, yielding x^2 - 2y^2 = -1 - a negative Pell equation. Pell equations are classical objects in algebraic number theory with a well-understood solution structure: if a fundamental solution exists, all solutions are generated by composing it repeatedly with the fundamental solution of the companion positive Pell equation x^2 - 2y^2 = 1. For this instance the composition gives the linear recurrence (x', y') = (3x + 4y, 2x + 3y), and each step scales n and b by a factor of roughly 3 + 2*sqrt(2), so the values grow exponentially fast.

Because growth is exponential, reaching a threshold of N requires only O(log N) recurrence steps. For the target of 10^12 this amounts to roughly a dozen iterations. The algorithm is therefore O(log N) in both time and space - essentially free from a computational standpoint, which is why every run reports zero elapsed time in the benchmark results.

Solution Approach

Pell Equation Recurrence Iteration (s0 - Python and C)

I implemented the same algorithm in both Python ( p0100_s0.py) and C ( p0100_s0.c). Starting from the seed (x, y) = (1, 1), which satisfies 1 - 2*1 = -1 and is the fundamental solution to the negative Pell equation, I iterate the recurrence (x', y') = (3x + 4y, 2x + 3y) at each step. After each step I recover n = (x + 1) / 2 and b = (y + 1) / 2 and halt as soon as n meets or exceeds the target threshold, returning b. The sequence of (n, b) pairs produced begins (1, 1), (3, 2), (21, 15), (120, 85), matching the examples given in the problem statement exactly.

The two implementations differ only in mechanics. In Python I use a simultaneous tuple assignment x, y = (3*x + 4*y, 2*x + 3*y), which evaluates both right-hand sides before any assignment and requires no temporary variable. In C, assignments are sequential, so I compute nx and ny into explicit temporaries before writing back to x and y. Both versions pass all three test cases - the named examples at n = 21 and n = 120 and the main target at n = 10^12 - and produce the correct answer of 756872327473.

Learnings

The key insight was recognising that the probability condition, after clearing denominators and rearranging, is a quadratic Diophantine equation, and that the centring substitution x = 2n - 1, y = 2b - 1 reveals a Pell equation in disguise. Without that substitution the equation looks like an intractable two-variable quadratic; with it, a three-line recurrence covers every solution up to any bound. Confirming the seed was a useful sanity check: (1, 1) satisfies the negative Pell equation and produces (n, b) = (1, 1) on the first recovery step, then (3, 2), so the loop generalises without needing to special-case any boundary condition.

Since only a single algorithm is implemented here there is nothing to compare between approaches. The Pell recurrence is already both maximally efficient and trivially short - there is simply nothing to improve on. The technique worth carrying forward to future problems is: whenever a combinatorial or probability condition produces a quadratic equation in two integer unknowns, look for a Pell equation in disguise. A centring substitution that eliminates linear terms is usually the right tool, and once the Pell form is exposed the solution reduces to iterating a fixed 2x2 integer matrix multiplication - or equivalently a pair of linear recurrences - until a threshold is crossed.
