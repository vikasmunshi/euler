Problem 123: Prime Square Remainders [Level 7]

Problem Statement

Let $p_n$ be the $n$th prime: $2, 3, 5, 7, 11, \dots$, and let $r$ be the remainder when $(p_n - 1)^n + (p_n + 1)^n$ is divided by $p_n^2$.

For example, when $n = 3$, $p_3 = 5$, and $4^3 + 6^3 = 280 \equiv 5 \mod 25$.

The least value of $n$ for which the remainder first exceeds $10^9$ is $7037$.

Find the least value of $n$ for which the remainder first exceeds $10^{10}$.

Mathematical Complexity

The core insight comes from applying the binomial theorem to expand (p_n - 1)^n and (p_n + 1)^n modulo p_n^2. Writing p for p_n, the expansion of (p+1)^n gives 1 + n*p + O(p^2) and that of (p-1)^n gives (-1)^n + n*(-1)^(n-1)*p + O(p^2). When n is even the constant terms sum to 2 and the linear terms cancel, so the remainder is always 2 - far too small to exceed any large threshold. When n is odd the constant terms cancel and the linear terms double, giving remainder r = 2*n*p_n (provided 2*n < p_n, which holds throughout the relevant range). The problem therefore reduces to finding the smallest odd index n for which 2*n*p_n exceeds the threshold, with no modular exponentiation required at all.

Generating all primes up to a fixed limit with a sieve of Eratosthenes costs O(L log log L) where L is the sieve bound. The subsequent linear scan over odd indices is O(N) where N is the answer index. For the thresholds tested here the answer index is in the tens of thousands, so the sieve dominates and the overall complexity is effectively O(L log log L).

Solution Approach

Closed-form scan after binomial reduction (s0, Python and C)

I implemented this single algorithm in both Python ( p0123_s0.py) and C ( p0123_s0.c). The steps are identical in both languages:

  • Run a sieve of Eratosthenes up to 1,000,000 to collect all primes in order.
  • Iterate over 1-indexed positions n = 1, 2, 3, .... Skip even n immediately, since the remainder is always 2 for even n.
  • For odd n, compute r = 2 * n * p_n using only integer arithmetic - no modular exponentiation, no big-integer arithmetic.
  • Return the first n for which r exceeds the threshold.

The only language-level difference is that in Python the threshold is read as int(argv[1]) passed through keyword arguments, while the C version reads it with atoll. Both use a plain long long -sized computation (Python integers are arbitrary precision, but the values involved are well within 64-bit range).

Learnings

The key unlock was recognising that the binomial theorem, applied mod p^2, collapses the entire expression to a two-case formula: the remainder is either 2 (even n) or exactly 2*n*p_n (odd n). What looks like a modular exponentiation problem turns into a direct inequality check. This kind of reduction - expanding a sum via the binomial theorem and discarding higher-order terms modulo a prime power - is a recurring technique in number-theory problems involving expressions of the form (p +/- 1)^n.

The C solution runs roughly 10 times faster than Python (around 3 ms versus 33 ms), consistent with the usual compiled-versus-interpreted gap on sieve-dominated workloads. Both are fast enough to be entirely practical; the Python version completes in under 40 ms even for the largest test case. The C implementation is worth keeping for consistency with other problems where the speedup matters more.

A technique worth remembering: whenever a problem asks for a remainder involving (a-1)^n + (a+1)^n modulo a^2, reach for the binomial theorem first. The odd/even parity split is immediate, and in the odd case only the single linear term survives, turning an exponentiation into a multiplication.
