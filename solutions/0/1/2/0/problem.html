Problem 120: Square Remainders [Level 4]

Problem Statement

Let $r$ be the remainder when $(a - 1)^n + (a + 1)^n$ is divided by $a^2$.

For example, if $a = 7$ and $n = 3$, then $r = 42$: $6^3 + 8^3 = 728 \equiv 42 \mod 49$. And as $n$ varies, so too will $r$, but for $a = 7$ it turns out that $r_{\mathrm{max}} = 42$.

For $3 \le a \le 1000$, find $\sum r_{\mathrm{max}}$.

Mathematical Complexity

The core of this problem is recognising that the expression (a-1)^n + (a+1)^n mod a^2 has a closed form derivable through binomial expansion. Expanding each term via the binomial theorem and summing them, terms where (n-k) is odd cancel in pairs, while terms where (n-k) is even double. Working modulo a^2, only the k=0 and k=1 terms survive, giving a remainder of 2 when n is even, and 2na mod a^2 when n is odd. The maximisation then reduces to a simple modular arithmetic question: how large can 2n mod a become as n ranges over the odd positive integers?

The answer depends on the parity of a. When a is odd, gcd(2, a) = 1, so 2n mod a attains every residue from 0 to a-1, giving r_max = a(a-1). When a is even, 2n is always even, so the largest even residue below a is a-2, giving r_max = a(a-2). With this formula in hand the summation over the range 3 <= a <= 1000 is a single linear scan, making the overall complexity O(a).

Solution Approach

Closed-Form Linear Scan (s0 - Python and C)

I implemented the same algorithm in both Python ( p0120_s0.py) and C ( p0120_s0.c). Both iterate a from 3 to the supplied limit and accumulate the maximum remainder using the formula derived above:

  • If a is odd: add a * (a - 1)
  • If a is even: add a * (a - 2)

No modular exponentiation, no inner loop, and no table lookups are needed. The Python version documents the full binomial expansion derivation in the docstring. The C version is structurally identical but uses long long arithmetic to avoid overflow; the Python version benefits from arbitrary-precision integers so no such annotation is required. Both versions accept the upper limit as a command-line argument to support the dev, main, and extra test cases.

Learnings

The key insight that unlocked this problem was recognising that binomial expansion modulo a^2 is a powerful simplification tool. All terms with k >= 2 contain a factor of a^2 and vanish, leaving at most two surviving terms. This collapses what looks like a computationally expensive search over all n into a single arithmetic expression per value of a.

The secondary insight was the modular arithmetic argument about the range of 2n mod a. Whenever gcd(2, a) = 1 (i.e. a is odd), the multiplier 2 is invertible mod a and the full residue ring is accessible; otherwise only even residues are reachable. This parity split is a pattern worth remembering in problems involving maximising linear residues.

The benchmark results confirm that both implementations are effectively instantaneous for the main input. The C version completes in under a microsecond and the Python version in around 0.1 ms - the difference is purely constant-factor overhead from the Python interpreter, since the algorithm is O(a) with no inner work. For a problem at this level of mathematical simplicity, the Python solution is arguably more valuable because the docstring serves as a self-contained proof of the formula.
