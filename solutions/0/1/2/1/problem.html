Problem 121: Disc Game Prize Fund [Level 10]

Problem Statement

A bag contains one red disc and one blue disc. In a game of chance a player takes a disc at random and its colour is noted. After each turn the disc is returned to the bag, an extra red disc is added, and another disc is taken at random.

The player pays £1 to play and wins if they have taken more blue discs than red discs at the end of the game.

If the game is played for four turns, the probability of a player winning is exactly 11/120, and so the maximum prize fund the banker should allocate for winning in this game would be £10 before they would expect to incur a loss. Note that any payout will be a whole number of pounds and also includes the original £1 paid to play the game, so in the example given the player actually wins £9.

Find the maximum prize fund that should be allocated to a single game in which fifteen turns are played.

Mathematical Complexity

The core challenge is computing the exact probability that a player takes more blue discs than red over n turns. On turn i (0-indexed), the bag holds 1 blue and i+1 red discs, giving a total of i+2 discs. The probability of drawing blue on that turn is 1/(i+2) and red is (i+1)/(i+2). The product of all turn denominators is 2 * 3 * ... * (n+1) = (n+1)!, so the overall probability of any specific sequence of outcomes has this factorial as its denominator. Working in integer numerators over this common denominator avoids floating-point issues entirely. The player wins if the number of blue draws exceeds n/2, so I need the sum of numerators for all winning configurations. The maximum prize fund is then floor((n+1)! / favorable) - the largest whole number such that the expected payout does not exceed the £1 entry fee.

The dynamic programming recurrence runs over n turns, maintaining a state vector of size n+1 (one entry per possible blue count). Each turn updates every reachable state in O(n) work, giving an overall time complexity of O(n^2) and space complexity of O(n). For the problem's target of 15 turns this is trivially fast; even the extra test case at 25 turns finishes in under a millisecond.

Dynamic Programming on Numerator Products (s0 - Python and C)

I chose to work entirely with integer numerator products rather than rational fractions or floating-point probabilities. The DP array dp[j] accumulates the sum of all numerator products for paths that have drawn exactly j blue discs so far. Initially dp[0] = 1. At each turn i, two transitions fire for every reachable state j:

  • Draw red: multiply the current value by the red numerator i+1 and keep the blue count at j.
  • Draw blue: multiply by 1 (blue numerator) and increment the blue count to j+1.

After all turns, the denominator is (turns+1)!. I sum dp[j] for all j >= turns//2 + 1 to get the favorable numerator, then take floor((turns+1)! / favorable) as the prize fund. The Python implementation uses arbitrary precision integers natively. The C implementation uses __int128, which is wide enough because for 15 turns the factorial is 16! = 20,922,789,888,000 (fits in 64 bits), and for 25 turns the numerator products stay within 128-bit range.

Learnings

The key insight was recognising that working with integer numerators over the common denominator (n+1)! is cleaner and exact. Trying to track probabilities as floats would accumulate rounding error and make the floor division unreliable. The DP formulation - treating each turn's outcomes as polynomial multiplication steps - is a standard technique for exact probability counting over variable-probability Bernoulli trials, and it avoids enumerating all 2^n outcome sequences explicitly. The prize fund formula itself follows directly from the requirement that prize * P(win) <= 1, i.e., the bank's expected loss per game must not exceed the entry fee. This pattern - find the exact rational probability, then invert and floor - recurs in a number of probability-based Euler problems and is worth keeping in mind.
