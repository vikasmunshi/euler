# Security Policy

## Supported Versions

| Version | Supported          |
|---------| ------------------ |
| 0.1.x   | :white_check_mark: |
| 0.0.x   | :x:                |

## Reporting a Vulnerability

We take the security of our project seriously. If you discover a security vulnerability, please follow these steps to report it:

1. **Do not** disclose the vulnerability publicly until it has been addressed.
2. Email the project author directly at [vikas.munshi@gmail.com](mailto:vikas.munshi@gmail.com) with details about the vulnerability.
3. Include as much information as possible, such as:
   - A description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact of the vulnerability
   - Suggestions for addressing the issue (if any)

## What to Expect

As this is a hobby project, please understand that response times may vary based on availability:

- **Acknowledgment**: You will receive an acknowledgment of your report when it's reviewed.
- **Assessment**: The vulnerability will be investigated with care and thoroughness rather than speed.
- **Resolution Approach**: The focus will be on implementing the right solution, not the fastest one.
- **Resolution and Disclosure**: Once the vulnerability is fixed, you will be notified and given credit for your discovery if you wish.

## Security Updates

Security updates will be incorporated into the project at a pace consistent with a hobby project, prioritizing correctness over speed. Critical vulnerabilities may be addressed more promptly depending on their severity and impact.

Thank you for your understanding and for helping keep this project secure.
